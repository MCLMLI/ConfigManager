# ConfigManager 配置管理器

> **现代化的配置文件管理工具，让复杂的配置文件管理变得简单直观**

ConfigManager 是一款专为配置文件设计的可视化管理工具。无论您是开发者、运维人员，还是需要经常处理配置文件的用户，ConfigManager 都能帮您轻松管理各类配置文件，提升工作效率。

---

## 🌟 为什么选择 ConfigManager？

### 📂 统一管理，井然有序
告别散落各处的配置文件！使用**工作区**功能，将所有相关配置文件集中管理。无论是项目配置、服务器配置还是应用设置，都能一目了然。

### 👀 可视化展示，一看就懂
不再面对冗长的文本文件！ConfigManager 自动将配置文件解析为**清晰的树形结构**，层级关系、数据类型一眼即知。支持展开/收起，轻松查看大型配置文件。

### ✏️ 即点即改，简单高效
**无需手动编辑 JSON、YAML 等复杂语法**！直接点击就能修改键名和值，**智能识别**字符串、数字、布尔值等多种值类型并提供**便捷输入**，使用右键可以轻松转换值类型。点击旁边的按钮轻松添加或删除配置项。

### 🔄 双窗对比，差异清晰
左右双半区设计，可以同时打开两个配置文件进行**对比查看**。测试环境和生产环境的配置差异？一眼看清！

### ↩️ 撤销还原，安心编辑
担心改错？不用怕！支持**撤销/还原功能**（Ctrl+Z / Ctrl+Y），让您放心大胆地修改配置。

### 📦 批量导入，快速上手
支持**导入整个文件夹**，自动识别所有配置文件。可自定义文件后缀过滤规则，只导入您需要的文件类型。

## 📋 支持的文件格式

ConfigManager 支持业界主流的配置文件格式：

| 格式 | 文件扩展名 | 典型用途 |
|------|-----------|----------|
| **JSON** | `.json` | Web 应用、Node.js 配置 |
| **YAML** | `.yaml` `.yml` | Docker、Kubernetes、CI/CD |
| **TOML** | `.toml` | Rust、Python 项目配置 |
| **INI** | `.ini` `.conf` `.cfg` | 系统配置、桌面应用 |
| **Properties** | `.properties` | Java 应用配置 |
| **TXT** | `.txt` | 通用文本配置 |

## 📖 快速上手

### 第一步：启动应用

点击进入 Releases 页面，查看最新的 Releases 并下载里面的 `.exe` 或 `.msi` 文件。下载完毕后双击即可开始安装本软件，安装完毕后双击本软件即可启动应用

### 第二步：创建或打开工作区

**💡 什么是工作区？**  
工作区就像一个项目文件夹，用来集中管理一组相关的配置文件。例如，您可以为"生产环境配置"、"测试服务器"、"本地开发"分别创建不同的工作区。

**创建新工作区：**
1. 在欢迎页面输入工作区名称（如"我的项目配置"）
2. 点击 **"创建工作区"** 按钮
3. 选择工作区文件的保存位置（会生成一个 `.cmws` 文件）
4. 完成！现在可以开始导入配置文件了

**打开已有工作区：**
- 点击 **"选择工作区文件"** 选择 `.cmws` 文件
- 或者从页面底部的 **"最近打开"** 列表快速打开

### 第三步：导入配置文件

进入工作区后，您会看到左右两个配置池区域。

**导入单个文件：**
1. 点击左侧或右侧的 **"导入文件"** 按钮
2. 选择您的配置文件（JSON、YAML、TOML 等）
3. 文件会自动解析为树形结构显示

**批量导入文件夹：**
1. 点击 **"导入文件夹"** 按钮
2. 选择包含配置文件的文件夹
3. 程序会自动识别并导入所有支持的配置文件
4. 文件夹的目录结构会被保留

💡 **提示**：在设置中可以自定义要导入的文件后缀，避免导入不需要的文件。

### 第四步：查看和编辑配置

**查看配置：**
- 配置文件会以树形结构展示
- 点击节点前的箭头可以展开/收起
- 点击文件名旁的 **"展开全部"/"收起全部"** 快速操作

**编辑键名：**
1. **双击**配置项的键名
2. 输入新的名称
3. 按 **Enter** 保存，**Esc** 取消

**编辑值：**
1. **单击**配置项的值
2. 根据类型进行编辑：
   - 字符串：直接输入文本
   - 数字：输入数字
   - 布尔值：勾选框切换
   - null：显示为 null
3. 按 **Enter** 保存，**Esc** 取消

**转换数据类型：**
1. **右键点击**配置项的值
2. 选择 **"转换为..."** 菜单
3. 选择目标类型（字符串、数字、布尔值、null、对象、数组）

**添加/删除项：**
- **对象**：点击对象右侧的 **"+"** 添加属性，点击 **"×"** 删除属性
- **数组**：点击数组右侧的 **"+"** 添加元素，点击 **"×"** 删除元素

### 第五步：保存更改

**⚠️ 重要提示**：修改配置后不会立即保存到文件，您需要手动保存。

**保存方法：**
- 点击菜单栏 **"文件" → "保存工作区"**
- 或按快捷键 **Ctrl+S**
- 未保存的更改会有明显提示

**撤销和还原：**
- 改错了？按 **Ctrl+Z** 撤销
- 需要还原？按 **Ctrl+Y** 重做
- 支持最近 50 步操作记录

## ⚙️ 设置说明

点击菜单栏 **"设置" → "首选项"** 打开设置面板。

### 全局设置

适用于所有工作区的通用设置：

- **文件后缀过滤**：设置导入文件夹时自动识别的文件扩展名
  - 例如：`json,yaml,yml,toml,ini,conf`
  - 用英文逗号分隔多个后缀

- **关闭确认**：关闭应用前是否显示确认对话框

### 工作区专用设置

为当前工作区单独设置，优先级高于全局设置：

- **启用工作区专用设置**：勾选后，当前工作区将使用独立的设置
- **文件后缀过滤**：为当前工作区自定义文件类型规则

💡 **应用场景**：  
- 全局设置适合大多数情况
- 特殊项目需要导入不同类型的文件时，可启用工作区专用设置

## ❓ 常见问题

### 关于工作区

**Q: 什么是工作区文件？保存在哪里？**  
A: 工作区文件（`.cmws`）是 ConfigManager 用来记录您管理的配置文件列表和设置的文件。创建工作区时，您可以选择保存位置，建议放在项目文件夹中便于管理。

**Q: 一次可以打开多个工作区吗？**  
A: 当前版本一次只能打开一个工作区，但您可以在一个工作区中管理任意多个配置文件。需要切换项目时，可以通过"最近打开"列表快速切换。

### 关于编辑和保存

**Q: 修改配置后会立即保存到原文件吗？**  
A: **不会**。这是为了安全考虑，您需要手动保存（Ctrl+S）才会将更改写入原配置文件。未保存的更改会有明显提示，避免意外丢失。

**Q: 保存后原文件会保留格式吗？**  
A: 是的。ConfigManager 会尽可能保持原文件的格式和缩进风格。但某些特殊注释可能会丢失（这是解析器的限制）。

**Q: 误操作了怎么办？**  
A: 不用担心！按 Ctrl+Z 撤销操作。只要没有保存（Ctrl+S），原文件不会被修改。即使保存了，也可以撤销后重新保存。

### 关于文件管理

**Q: 支持哪些配置文件格式？**  
A: 支持 JSON、YAML、TOML、INI、TXT、Properties 等常见格式。详见上方"支持的文件格式"章节。

**Q: 如何从工作区删除文件？**  
A: 点击文件卡片右上角的"×"按钮，文件会从工作区移除，但**不会删除原文件**，您的数据是安全的。

**Q: 导入文件夹会把所有文件都导入吗？**  
A: 不会。只会导入符合"设置"中指定后缀的文件。您可以在全局设置或工作区设置中自定义需要导入的文件类型。

### 关于数据安全

**Q: ConfigManager 会上传我的配置数据吗？**  
A: **绝对不会**。ConfigManager 是纯本地运行的软件，不连接任何服务器，不收集、上传或传输任何数据。您的所有配置文件都保存在本地设备上。

**Q: 软件需要联网吗？**  
A: 不需要。ConfigManager 完全离线运行，所有功能均可在无网络环境下使用。
